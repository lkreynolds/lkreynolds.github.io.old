Random Stuff I've learned:

Variables

  • GIGO: Garbage in, garbage out!
  • Variables are containers.
  • JavaScript variable names can contain numbers, letters, dollar signs, and underscores. They cannot begin with a number.
  • JavaScript is case sensitive!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
  • You can create and define a variable like this: var name = "Lindsay";
  • You don't have to define a datatype for JavaScript variables
  • JavaScript is "Weakly-Typed"
  • Strings can be held between either double or single quotes.

Characters/Strings

  • How to 'escape quotes:' "She shouted, \"Goodbye!\" to me." (Use a backslash before the quotation marks.)
  • Variables of the string type are smart - there are methods that can be used with strings.
  • How to insert a line break within a string: \n

Operators

  • Multiplication: *, division: /, addition: +, subtraction: -
  • Operator Precedence: P E M D A S
  • Shorthand operators: +=, -=, *=, /=, ++ (increment, --
  • The addition operator becomes the concatenation operator when one tries to add two strings. If one tries to add a string number and a number, concatenation will take precedence
  • !: NOT operator, negates

Conditionals

  • Basic if statement: if() { }
  • Check for equailty: == or === (the latter is the STRICT equality operator, will return false if the values are equal, but not the same data type).
  • Don't use the assignment operator instead of the equailty operator!!!!
  • && : AND
  • || : OR
  • Alternative to if statement:

    switch (variable)
    {
    case "x": do x;
    break;
    case "y": do y;
    break;
    default: do z;
    }

  • The break statement prevents "fallthrough," it "breaks" out of the switch statement.

Functions

  • Create: function functionName(parameters){function stuff; return result;}
  • Call: functionName(parameters);
  • Best Practice: Define functions before you call them. (JavaScript does an initial "scan" for functions in the whole file, so it doesn't really matter where you define functions, though.)
  • Other word for function: module, subroutine, method
  • Simple definition for recursion: when a function calls itself at least once
  • Scope: Where an object is visible
  • Local variables have local scope.
  • Global variables have global scope.
  • Variables declared within a function have scope within that function; the same for loops is not true (at least with javascript)
  • If you put a group of functions into different files, make sure the references to those files are listed in order of dependence.

Iterators

  • Simple loop:
    while(condition){
    do x;
    }
  • Be sure to include an index, and manupilate it to avoid an infinite loop.
  • Another simple loop:
    for(i=0; i < 10; i++){
    do x;
    }
  • Another simple loop:
    do{
    x;
    } while(condition);

A few built in functions

  • isNaN() : returns TRUE if the variable is not a number, false otherwise
  • Number() : converts the argument to a number
  • Built in string methods
    • string.toUpperCase()
    • string.toLowerCase()
    • string.indexOf("search"): if the "search" string exists within the string, return the index of where it begins; returns -1 if not
    • string.lastIndexof("search"):returns the last index location of the search string if it exists, -1 if not
    • string.slice(iBegin, iEnd): returns every character between iBegin and iEnd
    • string.substring()
    • string.substr()
    • Note: When comparing two strings, uppercase letters are regarded is less than lowercase letters

Regular expressions

  • create regular expressions: var regExp = /hello/;
  • or, this way: var my regExp = new RegExp("hello");
  • built in method: regExp.test(string): tests if the string contains the regular expression help in the regExp variable, returns a boolean
  • /^hello/ : carrot means at the start
  • /hello$/ : dollar sign means at the end
  • /hel+o/ : plus sign means l appears once or more
  • /hel*o/ : asterisk means l appearss zero or more
  • /hel?o/ : question mark means l appears zero or once
  • /hello|goodbye : pipe means either hello or goodbye appears
  • /he..o/ : decimal means any character
  • /\wello/: backslash and w means alphanumeric or underscore
  • /\bello/: backslash and b means word boundary: space or new line
  • /[clndr]ope/ :brackets indicate a range
  • Regular expressions are used to check for formatting: if the input matches the format of an e-mail address, phone number, cc number, etc

Arrays

  • Declare an array: var myArray = [];
  • Access an element in an array: myArray[0] = 1;
  • Load up array in shorthand method: myArray = [1, 2, 3];
  • Arrays are collections of data, but they are smart.
  • .length is a property that will return the length of an array.
  • Array Methods
    • .sort() : Will sort according to JavaScript sorting (alphabetically or numerically)
    • .join() : Will combine all elements into a single string, elements separated by commas
    • .pop() : will grab last element of the array, "pop" it off, and return its value
    • .push() : will push a value to the end of an array as at a new index
