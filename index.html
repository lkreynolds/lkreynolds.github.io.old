Random Stuff I've learned:

Variables

  • GIGO: Garbage in, garbage out!
  • Variables are containers.
  • JavaScript variable names can contain numbers, letters, dollar signs, and underscores. They cannot begin with a number.
  • JavaScript is case sensitive!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
  • You can create and define a variable like this: var name = "Lindsay";
  • You don't have to define a datatype for JavaScript variables
  • JavaScript is "Weakly-Typed"
  • Strings can be held between either double or single quotes.

Characters/Strings

  • How to 'escape quotes:' "She shouted, \"Goodbye!\" to me." (Use a backslash before the quotation marks.)
  • Variables of the string type are smart - there are methods that can be used with strings.
  • How to insert a line break within a string: \n

Arithmetic

  • Multiplication: *, division: /, addition: +, subtraction: -
  • Operator Precedence: P E M D A S
  • Shorthand operators: +=, -=, *=, /=, ++ (increment, --

Conditionals

  • Basic if statement: if() { }
  • Check for equailty: == or === (the latter is the STRICT equality operator, will return false if the values are equal, but not the same data type).
  • Don't use the assignment operator instead of the equailty operator!!!!
  • && : AND
  • || : OR
  • Alternative to if statement:

    switch (variable)
    { case "x": do x; break;
    case "y": do y; break;
    default: do z;}


    The break statement prevents "fallthrough," it "breaks" out of the switch statement.

Functions

  • Create: function functionName(parameters){function stuff; return result;}
  • Call: functionName(parameters);
  • Best Practice: Define functions before you call them. (JavaScript does an initial "scan" for functions in the whole file, so it doesn't really matter where you define functions, though.)
  • Other word for function: module, subroutine, method
  • Simple definition for recursion: when a function calls itself at least once
